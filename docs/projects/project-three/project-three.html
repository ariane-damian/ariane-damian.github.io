BERT vs Gen AI: Sentiment Analysis of Women’s E-commerce Clothing Reviews

MSc Business Analytics · NLP & Sentiment Analysis · Ariane Damian · Apr. 17, 2026

Introduction

Consumer reviews have long served as a cornerstone of business reputation and trust. In the digital age, the volume and velocity of online feedback has made manual analysis increasingly unsustainable, giving rise to the need for automated sentiment analysis.

This report compares two NLP models for binary sentiment classification of women’s e-commerce clothing reviews:

  • Evaluate BERT (encoder-based, discriminative) as a fine-tuned classification model
  • Evaluate LLaMA3 (decoder-based, generative) using zero-shot prompting
  • Assess both models across key performance metrics: accuracy, F1-score, precision, and recall
  • Discuss business relevance, ethical considerations, and limitations for e-commerce deployment

Methodology

Model Selection

Two transformer-based NLP models were selected to reflect diferent approaches to Natural Language Understanding tasks:

Model Type Approach
BERT Encoder-based (discriminative) Fine-tuned on labelled dataset via supervised learning
LLaMA3 Decoder-based (generative) Zero-shot prompting — no training or labelled examples required

Dataset

  • Source: Kaggle — real-world women’s e-commerce clothing reviews
  • ~23,500 records and 10 attributes prior to cleaning
  • Target variable: Recommended IND (binary: 1 = Recommended, 0 = Not Recommended)
  • Sentiment distribution: 82% Recommended / 18% Not Recommended (imbalanced)

Performance Metrics

Accuracy, F1-score, precision, and recall were all assessed. Due to the class imbalance, accuracy alone is insufficient — it risks biasing the model toward the majority class, violating the EU’s Trustworthy AI requirement of Fairness. F1-score was prioritised as the balance between precision and recall.

#code-fold: show
import matplotlib.pyplot as plt
import numpy as np

np.random.seed(42)

# ── Helper: simulate batch loss with smooth rolling average ──────────────────
def make_losses(n_batches, start, end, noise=0.18):
    trend = np.linspace(start, end, n_batches)
    noise_arr = np.random.normal(0, noise, n_batches)
    batch_loss = np.clip(trend + noise_arr, 0.05, 1.2)
    rolling = np.convolve(batch_loss, np.ones(10)/10, mode='valid')
    return batch_loss, rolling

# ── Fig 3.11a — 1 Epoch (100 batches) ───────────────────────────────────────
batch_loss_1, rolling_1 = make_losses(100, start=0.6186, end=0.3705)

fig, ax = plt.subplots(figsize=(8, 4))
ax.plot(batch_loss_1, color='#7ec8e3', linewidth=0.9, label='Batch loss')
ax.plot(range(9, 100), rolling_1, color='red', linewidth=2, label='10-batch rolling avg')
ax.set_title('Training Loss per Batch — 1 Epoch')
ax.set_xlabel('Batch number')
ax.set_ylabel('Loss')
ax.legend(fontsize=9)
ax.text(0.01, 0.05,
        f'Starting loss:  0.6186\nEnding loss:    0.3705\nLoss reduction: 0.2481',
        transform=ax.transAxes, fontsize=8, verticalalignment='bottom',
        fontfamily='monospace')
plt.tight_layout()
plt.savefig('fig_3_11a.png', dpi=150, bbox_inches='tight')
plt.show()
print("Note: Batch loss values are simulated to reflect reported statistics.")

# ── Fig 3.11b — 3 Epochs (300 batches) ──────────────────────────────────────
# Each epoch: loss resets slightly higher then trends down again
b1, r1 = make_losses(100, start=0.5604, end=0.25)
b2, r2 = make_losses(100, start=0.30,   end=0.12)
b3, r3 = make_losses(100, start=0.15,   end=0.0174, noise=0.08)

Discussion

Business Relevance

  • For e-commerce organisations, LLaMA3 is the more accessible option — requiring no data labelling, training infrastructure, or deep technical knowledge
  • However, BERT is substantially more reliable at detecting genuine negative reviews, which carries real operational risk if misclassified (e.g. misdirected resources, unnecessary escalations)
  • A practical compromise is DistilBERT — a derivative of BERT operating at 97% of its capacity, 60% faster and 40% smaller — prioritising rapid deployment over exhaustive complexity

Ethics & Responsible AI

  • The dataset is anonymised and from an authentic source — consistent with EU Trustworthy AI requirements under Privacy and Data Governance
  • Class imbalance introduces bias risk, particularly violating the Fairness principle if the minority class is under-represented in training
  • LLaMA3’s zero-shot bias toward negative classification raises additional concerns around reliability in production environments

Recommendations

The following directions are recommended for future work and deployment considerations:

  • Evaluate DistilBERT as a production-ready alternative to BERT — nearly equivalent performance at significantly lower resource cost
  • Explore few-shot prompting with LLaMA3 to assess whether providing examples improves precision for the minority class
  • Conduct hyperparameter tuning on BERT (learning rate, batch size, sample size) to address remaining volatility at epoch 3
  • Establish a control group (no class weights) to formally measure the impact of class weighting on model performance
  • Extend to multi-class sentiment classification to explore whether the metric uniformity observed in weighted averages persists across more than two labels
Note

BERT demonstrated stronger and more balanced performance for the binary classification objective, with higher accuracy and more reliable detection of negative sentiment. LLaMA3’s zero-shot results, while slightly weaker, highlight the growing potential of generative models for classification tasks — particularly in resource-constrained or low-expertise environments.

### Data Preparation & Preprocessing

- Null values in *Review Text* (845 total) were removed as this column functions as the model input
- A stratified sample of **2,000 records** was extracted to preserve class distribution
- Train/test split: **1,600 training / 400 test reviews** (80/20, random seed 42)

- BERT tokenisation used WordPiece Tokenizer (30,522 tokens); truncation threshold set at **128 tokens** based on median review length analysis

### Model Training & Prompting

- **BERT** was fine-tuned using an **AdamW optimiser** (corrects weight decay issues present in standard Adam) at a learning rate of 2e-5 and batch size of 16, trained over **3 epochs**
- **LLaMA3** was evaluated locally via zero-shot prompting using a structured prompt with three elements:

| Prompt Element | Content |

| **Instruction** | Classify the review as either RECOMMENDED or NOT RECOMMENDED |
| **Constraint** | Only reply with one of those two labels, nothing else |

### Performance Metrics

Accuracy, F1-score, precision, and recall were all assessed. ## Discussion

### Business Relevance

- For e-commerce organisations, **LLaMA3** is the more accessible option — requiring no data labelling, training infrastructure, or deep technical knowledge

## Recommendations
